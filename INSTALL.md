# Инструкция по установке и настройке

## Требования

- Python 3.8 или выше
- Discord Bot Token
- Права администратора на сервере Discord

## Установка

1. **Клонируйте репозиторий или скачайте файлы**

2. **Установите зависимости:**
```bash
pip install -r requirements.txt
```

3. **Настройте config.json:**
   - Откройте `config.json`
   - Замените `YOUR_BOT_TOKEN_HERE` на токен вашего бота
   - Настройте ID ролей и каналов под ваш сервер
   - Настройте параметры защиты по необходимости

4. **Настройте права бота:**
   Бот должен иметь следующие права:
   - Управление ролями
   - Управление каналами
   - Управление сообщениями
   - Кикать участников
   - Банить участников
   - Просмотр журнала аудита
   - Управление вебхуками
   - Чтение истории сообщений

5. **Запустите бота:**
```bash
python main.py
```

## Настройка сервера

### Создание ролей

1. **Роль верифицированного пользователя:**
   - Создайте роль с названием на ваше усмотрение
   - Скопируйте ID роли и вставьте в `config.json` → `roles.verified`

2. **Роль непроверенного пользователя:**
   - Создайте роль для непроверенных пользователей
   - Скопируйте ID роли и вставьте в `config.json` → `roles.unverified`
   - Настройте права этой роли так, чтобы пользователи не могли видеть основные каналы

3. **Роль администратора:**
   - Создайте роль для администраторов
   - Скопируйте ID роли и вставьте в `config.json` → `roles.admin`

### Создание каналов

1. **Канал верификации:**
   - Создайте канал для верификации
   - Скопируйте ID канала и вставьте в `config.json` → `channels.verification`
   - Бот автоматически отправит эмбед верификации при запуске

2. **Каналы логов:**
   - Создайте каналы для логов:
     - Канал для логов верификации
     - Канал для логов присоединений
     - Канал для логов безопасности
     - Канал для логов модерации
   - Скопируйте ID каналов и вставьте в соответствующие поля `config.json` → `channels`

## Команды

### Префиксные команды

- `!addl <bot_id>` - Добавить бота в whitelist (только для администраторов)
- `!logs` - Показать информацию о системе логирования

### Slash-команды

- `/mute <member> [duration] [reason]` - Заглушить пользователя
- `/unmute <member>` - Снять мут
- `/kick <member> [reason]` - Исключить пользователя
- `/ban <member> [reason] [delete_days]` - Забанить пользователя
- `/warn <member> [reason]` - Выдать предупреждение
- `/unwarn <member> [warning_id]` - Снять предупреждение
- `/clear <amount>` - Очистить сообщения
- `/slowmode <seconds>` - Установить медленный режим
- `/lock` - Заблокировать канал
- `/unlock` - Разблокировать канал

## Системы защиты

### Включение/выключение модулей

Все модули можно включить/выключить в `config.json`:

```json
{
  "security": {
    "anti_bot": {
      "enabled": true
    },
    "anti_nuke": {
      "enabled": true
    },
    "anti_spam": {
      "enabled": true
    }
  }
}
```

### Настройка параметров

Все параметры защиты настраиваются в `config.json`. Например:

```json
{
  "security": {
    "anti_spam": {
      "enabled": true,
      "max_messages": 5,
      "time_window": 5,
      "mute_duration": 300
    }
  }
}
```

## Устранение неполадок

### Бот не отвечает на команды

- Проверьте, что токен бота правильный
- Убедитесь, что бот имеет необходимые права
- Проверьте, что включены необходимые Intents в Discord Developer Portal

### Верификация не работает

- Проверьте, что ID канала верификации правильный
- Убедитесь, что бот может отправлять сообщения в канал
- Проверьте права бота на управление ролями

### Ошибки базы данных

- Удалите файл `bot.db` и перезапустите бота
- Убедитесь, что у бота есть права на создание файлов

## Поддержка

При возникновении проблем проверьте:
1. Логи бота в консоли
2. Права бота на сервере
3. Правильность настроек в `config.json`
4. Версию Python (должна быть 3.8+)
